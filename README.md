### How I think about building things

I start from the system, not the framework. Data flow, failure modes, what breaks at scale — then pick the tools.

On AI projects, I go past the "call the API, return the text" stage. I build retrieval pipelines with structured outputs, agents that take real actions, and AI features that live inside production apps with auth, rate limits, and error handling.

On mobile, I build once in Flutter. On web, Next.js with a real backend — not client components duct-taped to a serverless function.
